In this tutorial, you'll build a complete OTP (one-time password) login flow from scratch using Duende User Management. By the end, you'll have a working ASP.NET Core Razor Pages app where users log in with their email address and a one-time code.

<Aside type="caution">
  This guide uses in-memory storage. All data is lost when the app restarts, so it is intended for local development only.
  For production, switch to a persistent storage engine such as PostgreSQL or SQL Server. See the [Storage documentation](/usermanagement/fundamentals/storage.md) for details.
</Aside>

## Prerequisites

* .NET 10 SDK or later
* A code editor (e.g. Visual Studio, VS Code, JetBrains Rider)

## Create the Project

Run the following commands to scaffold a new ASP.NET Core web app and move into its directory:

```bash title="Terminal"
dotnet new webapp -o OtpUserManagement && cd OtpUserManagement
```

The `webapp` template gives you a Razor Pages project with a `Program.cs` entry point, a `Pages/` folder with a sample `Index` page, and the standard `appsettings.json` configuration files: everything you need to start building.

## Add NuGet Packages

Add the Duende User Management package:

```bash title="Terminal"
dotnet add package Duende.UserManagement
```

## Configure Services

Open `Program.cs` and modify its contents step by step as described below.

<Steps>

1. **Register cookie authentication**

   Register cookie authentication as the default scheme and configure the session cookie with secure defaults:

   ```csharp title="Program.cs"
   builder.Services
       .AddAuthentication(options =>
       {
           options.DefaultScheme = CookieAuthenticationDefaults.AuthenticationScheme;
           options.DefaultSignInScheme = CookieAuthenticationDefaults.AuthenticationScheme;
           options.DefaultAuthenticateScheme = CookieAuthenticationDefaults.AuthenticationScheme;
           options.DefaultChallengeScheme = CookieAuthenticationDefaults.AuthenticationScheme;
       })
       .AddCookie(CookieAuthenticationDefaults.AuthenticationScheme, options =>
       {
           options.Cookie.Name = "__Host-AuthSession";
           options.Cookie.HttpOnly = true;
           options.Cookie.SecurePolicy = CookieSecurePolicy.Always;
           options.Cookie.SameSite = SameSiteMode.Lax;
           options.Cookie.IsEssential = true;
           options.ExpireTimeSpan = TimeSpan.FromHours(8);
           options.SlidingExpiration = true;
           options.LoginPath = "/Login";
           options.LogoutPath = "/Logout";
       });
   ```

2. **Register Duende User Management services**

   Register the Duende User Management services, including the delivery mechanism for One-Time Passwords (OTP).

   For the application you are building, an OTP code will be required to log in. 
   These codes will need to be sent to your user so they can authenticate to your application.

   User Management contains an SMTP-based OTP sender, but you may choose other delivery mechanisms for your application.
   In this guide, we'll offer a (custom) console-based OTP sender, and demonstrate how to use the SMTP sender.

   <Tabs syncKey="umgsOtpSender">
     <TabItem label="Console sender (development)">
       The console sender prints OTP codes to the terminal, useful during local development without any mail infrastructure.

       ```csharp title="Program.cs"
       builder.Services
           .AddDuendePlatform()
           .AddUserAuthentication()
           .AddStorage(s => s.InMemory());
       ```

       User Management does not contain a console-based OTP sender. Implement `IOtpSender` to write codes to the console:
   
       ```csharp title="ConsoleOtpSender.cs"
       using Duende.Platform.Users.Authentication.Otp;

       public class ConsoleOtpSender : IOtpSender
       {
            public bool CanSend(OtpAddress address)
            {
                return true;
            }

            public Task SendAsync(OtpAddress address, PlainTextOtp otp, TimeSpan expiresAfter, CancellationToken ct)
            {
                Console.WriteLine($"OTP for {address}: {otp.Text}");
                return Task.CompletedTask;
            }
       }
       ```
   
       Register it as the default `IOtpSender` service, so User Management can use it:
   
       ```csharp title="Program.cs"
       builder.Services.AddSingleton<IOtpSender, ConsoleOtpSender>();
       ```
     </TabItem>
     <TabItem label="SMTP sender">
       The SMTP sender delivers OTP codes by email. Bind the `Smtp` configuration section to the sender options:

       ```csharp title="Program.cs"
       builder.Services
           .AddDuendePlatform()
           .AddUserAuthentication(authentication => authentication
               .UseSmtpOtpSender(options =>
                   builder.Configuration.GetSection("Smtp").Bind(options)))
           .AddStorage(s => s.InMemory());
       ```

       Add the corresponding section to `appsettings.json`:

       ```json title="appsettings.json"
       {
         "Smtp": {
           "Host": "localhost",
           "Port": 1025,
           "FromEmail": "noreply@example.com",
           "FromName": "noreply"
         }
       }
       ```

       :::tip[Development SMTP server]
       If you prefer an SMTP experience locally, [Mailpit](https://github.com/axllent/mailpit) is a lightweight local SMTP server that captures outgoing email in a web UI. No external mail account required.
       :::
     </TabItem>
   </Tabs>

3. **Register Data Protection**

   Register ASP.NET Core Data Protection so that cookies and tokens are encrypted at rest:

   ```csharp title="Program.cs"
   builder.Services.AddDataProtection();
   ```
   
   :::tip
   For production environments, configure [ASP.NET Core Data Protection](/general/data-protection.md) to persist encryption keys across restarts, for example using Azure Key Vault, Redis, or a shared file system.
   :::

4. **Add authentication middleware**

   Add the authentication and authorization middleware in the correct order:

   ```csharp title="Program.cs"
   // ...
   
   var app = builder.Build();
   app.UseAuthentication();
   app.UseAuthorization();
   ```

</Steps>

The application configuration is now complete. Next, we'll add the pages and models that will be used to handle the login, OTP verification, and logout flows.

## Add a Login Page

<Steps>
1. Create `Pages/Login.cshtml` with an email input form:
   
   ```html title="Pages/Login.cshtml"
   @page
   @model OtpUserManagement.Pages.LoginModel
   @{
       ViewData["Title"] = "Log in";
   }
   
   <h2>Log in</h2>
   
   <div asp-validation-summary="ModelOnly" class="validation-summary-errors"></div>
   
   <form method="post">
       <div>
           <label asp-for="Input.Email">Email address</label>
           <input asp-for="Input.Email" type="email" required autofocus autocomplete="email" />
           <span asp-validation-for="Input.Email"></span>
       </div>
       <button type="submit">Send one-time password</button>
   </form>
   ```
   
2. Then create the page model in `Pages/Login.cshtml.cs`:
   
   ```csharp title="Pages/Login.cshtml.cs"
   using System.ComponentModel.DataAnnotations;
   using Duende.Platform.Users;
   using Duende.Platform.Users.Authentication.Otp;
   using Microsoft.AspNetCore.Mvc;
   using Microsoft.AspNetCore.Mvc.RazorPages;
   
   namespace OtpUserManagement.Pages;
   
   public class LoginModel(IOtpAuthenticator otpAuthenticator) : PageModel
   {
       [BindProperty]
       public InputModel Input { get; set; } = new();
   
       public class InputModel
       {
           [Required]
           [EmailAddress]
           public string Email { get; set; } = string.Empty;
       }
   
       public void OnGet() { }
   
       public async Task<IActionResult> OnPostAsync()
       {
           if (!ModelState.IsValid)
           {
               return Page();
           }
   
           if (!EmailAddress.TryParse(Input.Email, out var email))
           {
               ModelState.AddModelError(nameof(Input.Email), "Invalid email format.");
               return Page();
           }
   
           var result = await otpAuthenticator.TrySendOtpAsync(
               new OtpAddress(OtpChannel.Email, email),
               HttpContext.RequestAborted);
   
           if (result is null || !result.Sent)
           {
               var message = "Failed to send one-time password.";
               if (result?.SendingBlockedUntilUtc is DateTimeOffset blockedUntil && blockedUntil > DateTimeOffset.UtcNow)
               {
                   var blockedFor = blockedUntil - DateTimeOffset.UtcNow;
                   message = $"Too many attempts. Try again in {Math.Ceiling(blockedFor.TotalSeconds)} second(s).";
               }
               ModelState.AddModelError(string.Empty, message);
               return Page();
           }
   
           TempData["OtpToken"] = result.Token.Value.ToString();
           return RedirectToPage("/EnterOtp");
       }
   }
   ```
</Steps>

When the form is submitted, `IOtpAuthenticator.TrySendOtpAsync` sends a one-time password to the provided email address and returns a result containing the OTP token.
If sending succeeds, the token is stored in `TempData` so the next page can verify the code the user enters. If sending is blocked due to rate limiting, a descriptive error is shown instead.

:::note
You can configure OTP expiry, code length, and rate-limiting behaviour through the `OtpAuthenticatorOptions`. See the [OTP authentication docs](/usermanagement/authentication/otp.mdx) for all available options.
:::

## Add an OTP Verification Page

<Steps>
1. Create `Pages/EnterOtp.cshtml`, the page where the user enters the one-time password they received by email:
   
   ```html title="Pages/EnterOtp.cshtml"
   @page
   @model OtpUserManagement.Pages.EnterOtpModel
   
   <h2>Enter one-time password</h2>
   
   @if (!ViewData.ModelState.IsValid)
   {
       <ul>
           @foreach (var error in ViewData.ModelState.Values.SelectMany(v => v.Errors))
           {
               <li>@error.ErrorMessage</li>
           }
       </ul>
   }
   
   <form method="post">
       <input type="hidden" asp-for="Input.Token" />
       <label asp-for="Input.Code">One-time password</label>
       <input asp-for="Input.Code" type="text" autocomplete="one-time-code" autofocus />
       <button type="submit">Sign in</button>
   </form>
   ```
   
2. Now create `Pages/EnterOtp.cshtml.cs`:
   
   ```csharp title="Pages/EnterOtp.cshtml.cs"
   using System.ComponentModel.DataAnnotations;
   using System.Security.Claims;
   using Duende.Platform.Users;
   using Duende.Platform.Users.Authentication;
   using Duende.Platform.Users.Authentication.Otp;
   using Microsoft.AspNetCore.Authentication;
   using Microsoft.AspNetCore.Authentication.Cookies;
   using Microsoft.AspNetCore.Mvc;
   using Microsoft.AspNetCore.Mvc.RazorPages;
   
   namespace OtpUserManagement.Pages;
   
   public class EnterOtpModel(
       IOtpAuthenticator otpAuthenticator,
       IUserAuthenticatorsSelfService authenticatorsSelfService)
       : PageModel
   {
       [BindProperty]
       public InputModel Input { get; set; } = new();
   
       public class InputModel
       {
           [Required]
           public string Token { get; set; } = string.Empty;
   
           [Required]
           public string Code { get; set; } = string.Empty;
       }
   
       public IActionResult OnGet()
       {
           var token = TempData["OtpToken"]?.ToString();
           if (token is null)
           {
               return RedirectToPage("/Login");
           }
   
           Input.Token = token;
           return Page();
       }
   
       public async Task<IActionResult> OnPostAsync()
       {
           if (!ModelState.IsValid)
           {
               return Page();
           }
   
           if (string.IsNullOrWhiteSpace(Input.Code) || string.IsNullOrWhiteSpace(Input.Token))
           {
               ModelState.AddModelError(string.Empty, "Invalid input.");
               return Page();
           }
   
           var otp = PlainTextOtp.Parse(Input.Code);
           var token = OtpToken.Parse(Input.Token);
   
           var address = await otpAuthenticator.TryAuthenticateAsync(
               otp, token, HttpContext.RequestAborted);
   
           if (address is null)
           {
               ModelState.AddModelError(string.Empty, "Invalid or expired code. Please try again.");
               return Page();
           }
   
           // Get the existing user record, or register a new one on first sign-in.
           var authenticators =
               await authenticatorsSelfService.TryGetAsync(address, HttpContext.RequestAborted) ??
               await authenticatorsSelfService.TryRegisterAsync(
                   UserSubjectId.New(), address, HttpContext.RequestAborted);
   
           if (authenticators is null)
           {
               ModelState.AddModelError(string.Empty, "Failed to sign in. Please try again.");
               return Page();
           }
   
           var claims = new List<Claim>
           {
               new(ClaimTypes.NameIdentifier, authenticators.SubjectId.ToGuid().ToString()),
               new(ClaimTypes.Name, address.SubjectId.ToString()),
           };
   
           var identity = new ClaimsIdentity(
               claims, CookieAuthenticationDefaults.AuthenticationScheme);
   
           await HttpContext.SignInAsync(new ClaimsPrincipal(identity));
   
           return LocalRedirect(Url.Content("~/"));
       }
   }
   ```
</Steps>

The code for this page handles both `GET` and `POST` requests, mapped to Razor Pages methods:

**`OnGet`** reads the OTP token that the Login page stored in `TempData["OtpToken"]` and copies it into the hidden form field. If the token is missing (e.g. the user navigated here directly), the page redirects back to `/Login`.

**`OnPostAsync`** parses the submitted token and code into their strongly-typed counterparts (`OtpToken` and `PlainTextOtp`), then calls `IOtpAuthenticator.TryAuthenticateAsync`. If the code is correct and not expired, the method returns the `OtpAddress` that was authenticated. A `null` return value means the code was wrong or has expired.

:::note[Automatic User Registration]
After a successful OTP verification, the page calls `IUserAuthenticatorsSelfService.TryGetAsync` to look up an existing user by their OTP address. If no record is found (which happens the very first time a given email address signs in), `TryRegisterAsync` is called instead to create a new user automatically. This "get-or-register" pattern means you do not need a separate sign-up flow: every new email address that successfully verifies an OTP becomes a registered user on the spot.

Once a user record is confirmed (or created), a `ClaimsPrincipal` is built from the subject ID and email address and passed to `HttpContext.SignInAsync`, which issues the authentication cookie and completes the sign-in.
:::

## Add a Logout Page

<Steps>
1. Create `Pages/Logout.cshtml`:
   
   ```html title="Pages/Logout.cshtml"
   @page
   @model OtpUserManagement.Pages.LogoutModel
   ```
   
2. Create `Pages/Logout.cshtml.cs`:
   
   ```csharp title="Pages/Logout.cshtml.cs"
    using Microsoft.AspNetCore.Authentication;
    using Microsoft.AspNetCore.Mvc;
    using Microsoft.AspNetCore.Mvc.RazorPages;

    namespace OtpUserManagement.Pages;
   
    public class LogoutModel : PageModel
    {
        public async Task<IActionResult> OnPostAsync()
        {
            await HttpContext.SignOutAsync();
            return RedirectToPage("/Login");
        }
    }
   ```
</Steps>

`OnPostAsync` calls `HttpContext.SignOutAsync()` to clear the authentication cookie, then redirects the user back to `/Login`. Using POST (rather than GET) prevents cross-site request forgery attacks that could silently sign a user out by embedding a link.

## Protect the Home Page

<Steps>
1. Update `Pages/Index.cshtml.cs` to require an authenticated user by adding the `[Authorize]` attribute:
   
   ```csharp title="Pages/Index.cshtml.cs" {6}
   using Microsoft.AspNetCore.Authorization;
   using Microsoft.AspNetCore.Mvc.RazorPages;
   
   namespace OtpUserManagement.Pages;
   
   [Authorize]
   public class IndexModel : PageModel
   {
       public void OnGet()
       {
       }
   }
   ```
   
2. Update `Pages/Index.cshtml` to greet the signed-in user and provide a sign-out button:
   
   ```html title="Pages/Index.cshtml"
   @page
   @model OtpUserManagement.Pages.IndexModel
   @{
       ViewData["Title"] = "Home";
   }
   
   <h2>Welcome, @User.Identity!.Name!</h2>
   
   <form method="post" asp-page="/Logout">
       <button type="submit">Sign out</button>
   </form>
   ```
</Steps>

`User.Identity!.Name` contains the email address that was set as `ClaimTypes.Name` during OTP verification.

Because `LoginPath = "/Login"` was configured on the cookie authentication options in `Program.cs`, any unauthenticated request to `/` is automatically redirected to the login page, with no extra middleware or filters required.

## Run and Test

<Steps>

1. Start the application:

   ```bash title="Terminal"
   dotnet run
   ```

2. Open your browser and navigate to `https://localhost:5001` (or the URL shown in the terminal).

3. You are redirected to `/Login`. Enter your email address and click **Send one-time password**.

4. Retrieve the OTP code:

   <Tabs syncKey="umgsOtpSender">
     <TabItem label="Console sender">
       The OTP code is printed directly to the terminal. Copy it from there.
     </TabItem>
     <TabItem label="SMTP sender">
       Check your inbox for the email containing the OTP code. If you are using Mailpit locally, open `http://localhost:8025` to find the message.
     </TabItem>
   </Tabs>

5. Enter the code on the `/EnterOtp` page and click **Sign in**.

6. You land on the home page showing a welcome message with the email address of the user.

7. Click **Sign out** to return to the login page.

</Steps>

**Congratulations!** You now have a working ASP.NET Core application with OTP-based authentication powered by Duende User Management. Users can sign in with their email address, verify a one-time code, and access protected pages. New users are automatically registered on their first successful login.
